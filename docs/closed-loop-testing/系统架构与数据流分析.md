## 1. 系统概述

Unitree G1 Agent 是一个基于 ROS2 的机器人控制系统，采用分层架构设计，确保所有运动指令都经过安全控制层。系统由四个主要节点组成：

- **g1_interface**: G1 原生接口桥接节点
- **safety_control**: 安全控制节点
- **voice_bridge**: 语音桥接节点  
- **g1_sim**: G1 原生主题仿真器

```python
SAFE_LOCO_LIMITS = {

# 状态超时检查 (node.py:162-170)
def check_sport_command_allowed(now_sec, last_lowstate_sec, state_timeout_sec):
    if last_lowstate_sec is None:
        return False, "lowstate unavailable"
    age_ms = int(round((now_sec - last_lowstate_sec) * 1000))
    if now_sec - last_lowstate_sec > state_timeout_sec:
        return False, f"lowstate stale: age_ms={age_ms}"
    return True, None
```

## 3. 数据流分析

### 3.1 语音行走的完整数据流

```mermaid
graph TD
    A[用户语音] --> B[ASR识别]
    B --> C[/g1/audio/asr]
    C --> D[voice_bridge节点]
    D --> E[Pi Agent处理]
    E --> F{/voice/cmd/loco}
    F --> G[safety_control节点]
    G --> H{安全检查}
    H -->|通过| I[/g1/safe_cmd/loco]
    H -->|拒绝| J[/g1/safety/decisions]
    I --> K[g1_interface节点]
    K --> L[/api/sport/request]
    L --> M[g1_sim/真实G1]
    M --> N[/api/sport/response]
    N --> K
    K --> O[/g1/state/low]
    O --> G
```

**详细步骤**:

1. **ASR 输入**: 用户语音 → ASR 服务 → `/g1/audio/asr`
2. **语音处理**: voice_bridge 接收 ASR → 会话管理 → Pi Agent 决策
3. **意图发布**: Pi Agent 输出 → `/voice/cmd/loco` (运动意图)
4. **安全验证**: safety_control 接收意图 → 多重检查 → 决策
5. **安全指令**: 通过检查 → `/g1/safe_cmd/loco` (安全指令)
6. **接口转换**: g1_interface 接收安全指令 → `/api/sport/request`
7. **执行反馈**: g1_sim/G1 执行 → `/api/sport/response` → 状态更新

### 3.2 紧急停止数据流

```mermaid
graph LR
    A[停止词] --> B[voice_bridge]
    B --> C[/voice/cmd/action]
    C --> D[safety_control]
    D --> E[/g1/safe_cmd/stop]
    E --> F[g1_interface]
    F --> G[/api/sport/request停止]
    G --> H[g1_sim/G1停止]
```

**关键特性**:
- 停止词绕过会话状态检查
- 紧急停止不受频率限制
- 直接发布到 `/g1/safe_cmd/stop`

### 3.3 状态监控数据流

```mermaid
graph TD
    A[g1_sim/G1硬件] --> B[lowstate]
    B --> C[g1_interface]
    C --> D[/g1/state/low]
    D --> E[safety_control]
    C --> F[/g1/state/mode]
    F --> E
    C --> G[/g1/state/health]
    G --> E
    E --> H{/g1/state/safety}
    H --> voice_bridge
```

## 5. 安全机制总结

### 5.1 多层安全防护

1. **g1_interface 层**:
   - 状态超时检测 (300ms)
   - 基本速度限制 (vx: ±0.5, vy: ±0.3, vyaw: ±0.8)
   - 持续时间限制 (0.01-2.0s)
   - API响应超时检测 (500ms)

2. **safety_control 层**:
   - 运动限幅检查
   - 模式门控
   - 频率限制 (令牌桶算法)
   - 速度连续性检查 (加速度、加加速度)
   - 状态新鲜度检查
   - 健康状态检查

3. **voice_bridge 层**:
   - 会话超时管理
   - ASR置信度检查
   - 紧急停止绕过机制
   - Agent故障保护

### 5.2 故障保护机制

| 故障类型 | 检测方式 | 保护动作 | 恢复条件 |
|---------|---------|---------|---------|
| 状态丢失 | lowstate超时 | 拒绝新指令 | 状态恢复 |
| 模式错误 | FSM mode检查 | 拒绝运动指令 | 模式切换 |
| 超速指令 | 速度限制检查 | 指令限幅或拒绝 | N/A |
| 高频指令 | 频率限制 | 丢弃指令 | 频率正常 |
| Agent故障 | 异步捕获 | 停止运动指令 | Agent恢复 |
| 节点崩溃 | ROS2检测 | 停止所有指令 | 节点重启 |
